### BasePawn 및 플레이어 캐릭터 클래스 계층 구현 및 리팩토링

**작업 내용:**

1.  **캐릭터 계층 구조 구현**: `APawn`을 직접 상속하는 `AZakaBasePawn`, `AZakaPlayerCharacter`, `AZakaZombiePawn` 및 `AZakaPlayerController`를 구현함.
2.  **폴더 구조 리팩토링**: Unreal Engine 표준 규정에 따라 전역 `Public/`, `Private/` 폴더 내에 카테고리별로 파일을 재배치함.

| 파일 경로 | 설명 |
|-----------|------|
| `Public/Characters/*.h` | 캐릭터 관련 헤더 파일 |
| `Private/Characters/*.cpp` | 캐릭터 관련 소스 파일 |
| `Public/Framework/*.h` | 프레임워크 관련 헤더 파일 |
| `Private/Framework/*.cpp` | 프레임워크 관련 소스 파일 |

**설계 결정:**

- **Unreal 표준 구조 준수**: `Public/[Category]/[Header].h` 및 `Private/[Category]/[Source].cpp` 구조를 적용하여 모듈 관리 효율성을 높임.
- **Include 경로 최적화**: `#include "Characters/Filename.h"` 형식을 사용하여 명확한 의존성 표현.
- **사용자 커스텀 반영**: 사용자가 수정한 Getter, `SetupInput()`, 입력 핸들러 제거 등의 변경사항을 모두 유지하며 리팩토링 진행.
